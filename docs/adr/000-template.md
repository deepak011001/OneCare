# ADR-000: Record Architecture Decisions

## Status

Accepted (template)

## Context

Significant technical choices for OneCare must be recorded so future contributors and Cursor agents understand *why* a path was chosen.

## Decision

Use Architecture Decision Records under `docs/adr/` with sequential numbering (`ADR-001-…md`).

## Consequences

- PRs that change architecture link an ADR
- `docs/ARCHITECTURE.md` stays the overview; ADRs capture debate and tradeoffs
